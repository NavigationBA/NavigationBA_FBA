    Topic 2.2 - R 代码运行的方式¶

    1. R 代码在本地与云端运行¶

    (1) 在本地运行 R 代码¶

    在本地运行 R 需要在自己的电脑上下载两个软件：

    • R：核心编程语言和环境：

      • 计算机是只能运行机器码的（0 或 1 的二进制指令），默认的设置中，计算机是看不懂 R 代码的
      • 那么这个 R 软件的功能就是将你编写的 R 代码翻译成计算机可以执行的机器码
      • 它就像一本字典，帮助你的计算机理解你在 R 中给出的指令

    • RStudio：一个集成开发环境（IDE），提供了一个用户友好的界面来编写和运行 R 代码

      • 我们把 R 软件比作是一个字典，但是仅仅有一本 R 字典是不够的
      • 你还需要一个软件，提供一个良好的工作空间来编写和运行你的 R 代码，也就是说你还需要一个软件来指导你如何使用这本字典
      • 这个软件就是 RStudio，它是最受欢迎的 R IDE 之一

    这两个软件都是免费的，它们的下载连接如下：

    • R: https://cran.r-project.org/mirrors.html
    • RStudio: https://posit.co/download/rstudio-desktop/

    安装完 R 和 RStudio 后，你可以打开 RStudio 并查看其界面：

    • 注意：R 软件下载下来后可能永远不需要打开，因为 RStudio 会自动调用 R 来运行你的代码，但是千万不能因为 R 软件不需要打开，就不安装 R 软件，或者把它卸载掉了，否则 RStudio 就无法运行你的 R 代码了

    • 然后，你可以点击 "New File" > "R Script" 来创建一个新的 R 脚本文件，在这里你可以编写和运行你的 R 代码：

    • 然后，这个界面就显示了完整的 RStudio 界面，其中有四个主要面板：

      1. 左上角：Source 面板：你在这里编写 R 代码
      2. 左下角：Console 面板：你可以在这里直接输入和运行 R 命令
      3. 右上角：Environment/History 面板：你可以在这里查看你创建的变量和数据框，以及你的命令历史
      4. 右下角：Files/Plots/Packages/Help 面板：你可以在这里管理文件、查看图表、管理包以及访问帮助文档

    (2) 在云端运行 R 代码¶

    Posit Cloud 是一个基于云的 RStudio 服务，允许你在云端编写和运行 R 代码：

    • Posit Cloud 提供了一个在线的 RStudio 环境，你可以通过浏览器访问
    • Posit Cloud 的官方网站是 https://posit.cloud/

    要使用 Posit Cloud，你需要创建一个账户并登录，然后你可以看到以下界面：

    Posit Cloud 为你提供了工作区，用于创建和管理你的 R 项目：

    • 在 "Your Workspaces" 标签下，你可以看到你创建的 R 项目
    • 你也可以点击 "New Workspace" 来创建一个新的工作区，在新的工作区来创建属于这个工作区的 R 项目

    在每个工作区中，你可以通过点击 "New Project" > "New RStudio Project" 来创建 R 项目：

    • 然后，你可以看到一个在线的 RStudio 界面，这个界面与本地计算机上的 RStudio 界面相同
    • 你可以以与本地 RStudio 界面相同的方式编写和运行你的 R 代码

    (3) 对比本地运行和云端运行 R 代码¶

    本地运行 R 代码：

    • 需要在自己的电脑上安装 R 和 RStudio，并定期更新到最新版本
    • 可以在不联网的情况下运行 R 代码
    • 使用自己本地的计算资源（CPU、内存）
    • 文件和项目存储在你的本地电脑上，没有办法从不同设备访问

    在 Posit Cloud 上运行 R 代码：

    • 不需要安装任何软件，并且云端会自动更新 R 和 RStudio 到最新版本，你不需要担心软件的安装和更新问题
    • 必须联网才能使用 Posit Cloud 来运行 R 代码
    • 计算资源由云服务器提供，通常要比本地资源弱一些，但是对于小型数据分析任务来说是足够的
    • 文件和项目存储在云端，可以从不同设备访问，而无需担心文件管理

    2. R 代码运行文件¶

    (1) 第一个 R 程序¶

    我们先来编写一个简单的 R 程序，来看看 R 代码的运行方式：

    • 首先，我们打开 RStudio，在 Source 面板中创建一个新的 R Script 文件，左上角点击 "New File" > "R Script"
    • 这时，我们会看到左上角出现一个新的 R Script 文件，我们在这个文件中输入以下 R 代码：
    2 + 2
    • 之后，我们用鼠标选中这行代码，然后点击代码块左侧的绿色运行按钮，就可以运行这行代码了，看到以下结果：
    [1] 4

    接着，我们再来尝试一下在 Console 面板中直接输入 R 代码来运行：

    • 我们在 Console 面板中直接输入 3 + 3，然后按回车键来运行这行代码

    • 代码运行成功后，我们会看到以下结果：

    [1] 6

    通过这个简单的例子，我们可以看到，在 R 中，我们可以通过两种方式来运行 R 代码：

    1. 在 Source 面板中的 R Script 文件中编写 R 代码，然后通过快捷键或菜单来运行这些代码
    2. 在 Console 面板中直接输入 R 代码来运行

    这两种运行方式的区别是：

    • 在 Console 面板中输入的代码是即时执行的，适合快速测试和交互式编程
    • 在 Source 面板中的脚本文件中编写的代码需要先保存，然后才能运行，适合编写较长的代码和项目
    • 事实上，只要是代码，就必须在 Console 中运行，在文件中 编写的代码事实上是被发送到 Console 中运行的

    (2) Rmd 与 Qmd 文件的使用¶

    除了这两种方式以外，R 还有两种很重要的运行代码的方式：

    • R Markdown 文件：R Markdown 文件是一种将 R 代码与文本、图表等内容结合在一起的文档格式，可以生成高质量的报告、演示文稿和网页
    • Quarto 文件：Quarto 文件是 R Markdown 的升级版，支持更多的编程语言和更强的文档生成能力

    事实上，这两种文件格式本质上是差不多的，都是将代码和文本结合在一起的文档格式：

    • 只不过 Quarto 文件是 R Markdown 文件的升级版，支持更多的功能和更强的文档生成能力
    • 因此，我们后续就主要以 Quarto 文件为主来进行 R 代码的编写和运行
    • 当然，其他课程可能需要大家使用 R Markdown 文件来进行代码的编写和运行，这些都是差不多的，大家不需要担心，主要是掌握这种将代码和文本结合在一起的文档格式的使用方法就可以了

    (3) Quarto 文件的使用¶

    (a) 在 Quarto 文件中编写 R 代码¶

    我们在当前工作目录下新建一个 Quarto 文件：

    • 在新建文件的界面中选择 Quarto Document，这个文件就是一个 Quarto 文件了
    • 在弹出的对话框中，我们输入文档标题，选择文档类型为 HTML Document，然后点击 Create 就可以创建一个新的 Quarto 文件了
    • 之后，我们可以在这个文件中编写 R 代码和文本内容

    在 Quarto 文件中，R 代码专门在 R 代码块中编写：

    • 它由三个反引号开始，以 {r} 开头，表示这是一个 R 代码块
    • 然后在代码块中编写 R 代码
    • 最后以三个反引号结束，表示这个代码块的结束

    如果要运行这个代码块：

    • 只需将光标放在这个代码块中，然后点击代码块左侧的绿色运行按钮，就可以运行这个代码块中的 R 代码了
    • 运行完成之后，结果会显示在代码块的下方
    • 而且大家可以发现，这个代码也是在 Console 中运行的，所以我们在 Console 中也可以看到这个代码块中的 R 代码被执行了

    其他的 R 代码编写和运行方式都是一样的：

    • 我们在 R Script 文件中编写 R 代码的方式和在 Quarto 文件中编写 R 代码的方式是一样的
    • 都是使用 R 语言的语法来编写 R 代码，代码会被移动到 Console 中运行，运行结果会显示在 Console 中
    • 并且在 Quarto 文件中也会保存运行结果

    (b) 在 Quarto 文件中编写文本内容¶

    在 Quarto 文件中，除了代码之外的内容都是文本内容：

    • 我们可以直接在 Quarto 文件中编写文本内容
    • 这些文本内容会被渲染成漂亮的格式，适合于撰写报告、演示文稿等
    • 并且代码与代码的运行结果也会被很好地展示在文本内容中，形成一个完整的文档

    在 Quarto 文件中，文字的编写分为以下几种形式：

    • 标题：使用 # 来表示标题，# 的数量表示标题的层级，比如 # 表示一级标题，## 表示二级标题，依此类推，总共可以有四级标题
    • 普通文本：直接输入文本内容即可，这些文本内容会被渲染成段落，注意段落之间需要空一行来分隔
    • 公式：使用 $...$ 来表示行内公式，使用 $$...$$ 来表示行间公式，这些公式使用 LaTeX 语法，会被渲染成漂亮的数学公式

    (c) Quarto 文件渲染成目标文件格式¶

    在 Quarto 文件中，我们编写好代码和文本内容之后，我们可以将这个 Quarto 文件渲染成我们需要的目标文件格式：

    • 只需点击 Quarto 文件上方的 Render 按钮，就可以将这个 Quarto 文件渲染成 HTML、PDF、Word 等格式的文件了
    • 渲染完成之后，我们就可以在当前工作目录下看到这个渲染好的文件了，这个文件就是我们最终的报告、演示文稿或者网页了
    • 要注意，在渲染文本内容的时候，代码块中的 R 代码也会被执行一次，所以如果代码块中的 R 代码有错误的话，渲染就会失败，因此我们在渲染之前需要确保代码块中的 R 代码是没有错误的，这样才能成功地渲染出目标文件格式的文档来
