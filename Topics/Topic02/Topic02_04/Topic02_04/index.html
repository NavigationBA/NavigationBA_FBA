    Topic 2.4 - R 中的数据类型¶

    在 R 中，我们基本就只会接触到以下几种数据类型：

    • 数值型（numeric）
    • 字符型（character）
    • 因子型（factor）
    • 日期型（Date）

    本节课我们重点讲解数值型以及相关操作，其他数据类型我们这里简单介绍，后续的进阶操作我们会在之后的课程中详细介绍。

    1. 数值型（numeric）¶

    数值型数据在 R 中又分为三种类型：

    • 整数（integer）与双精度数（double）：其实就是大家理解的整数和小数
    • 逻辑型（logical）：也就是布尔值，只有两种取值：TRUE 和 FALSE
    • 空值（NA）：表示缺失值，表示数据中没有值或者值不可用

    (1) 整数与双精度数¶

    (a) 整数与双精度数的定义¶

    在 R 中，基本上所有的数字都是双精度数

    • 要想创建一个整数，我们需要在数字后面加上 L，比如 1L 就是一个整数
    • 而不加 L 的数字默认就是双精度数了，比如 1、3.14、-2.718 就是一个双精度数
    • 我们可以定义以下这几个变量来看看它们的类型，在 R 中，我们可以使用 typeof() 函数来查看一个变量的数据类型：
    a <- 1L
b <- 1
c <- 3.14
d <- -2.718

typeof(a)
typeof(b)
typeof(c)
typeof(d)

    [1] "integer"
[1] "double"
[1] "double"
[1] "double"
    • 可以说，整数这个东西我们在 R 中是很少见的，大家有的时候见到一个数字后面有个 L，就知道它是一个整数就可以了
    • 其他时候基本上我们看到的数字都是双精度数了

    (b) 算数运算符¶

    整数与双精度数是支持使用基本的算数运算符进行运算的：

    • 基本的算数运算符包括：

      • 加法（+）
      • 减法（-）
      • 乘法（*）
      • 除法（/）
      • 取模（%%）
      • 整除（%/%）
      • 乘方（^）
      • 正号（+）
      • 负号（-）

    • 注意，R 中运算符之间是讲究优先级的：

      • 乘方的优先级最高
      • 其次是乘法和除法
      • 最后是加法和减法

    • 如果想要强调某个运算的优先级，我们可以使用括号来改变默认的优先级

    • 我们来看以下一些运算结果：

    1 + 2 * 3
(1 + 2) * 3
2^3 + 4
2^(3 + 4)
-3 + 5
+3 - 5

    [1] 7
[1] 9
[1] 12
[1] 128
[1] 2
[1] -2

    (c) 运算函数¶

    除了基本的算数运算符之外，R 还提供了很多内置的运算函数来进行各种各样的数学运算：

    • 例如：

      • sqrt(x)：计算 x 的平方根
      • log(x)：计算 x 的自然对数
      • log(x, base)：计算 x 的以 base 为底的对数
      • log10(x)：计算 x 的以 10 为底的对数
      • log2(x)：计算 x 的以 2 为底的对数
      • exp(x)：计算 e 的 x 次幂
      • abs(x)：计算 x 的绝对值
      • round(x, digits)：将 x 四舍五入到指定的小数位数
      • ceiling(x)：向上取整，返回大于或等于 x 的最小整数（例如 ceiling(3.2) 返回 4，ceiling(-3.2) 返回 -3）
      • floor(x)：向下取整，返回小于或等于 x 的最大整数（例如 floor(3.8) 返回 3，floor(-3.8) 返回 -4）
      • trunc(x)：截断，返回 x 的整数部分，不是四舍五入（例如 trunc(3.8) 返回 3，trunc(-3.8) 返回 -3）

    • 我们在以下的代码中来看看这些函数的使用示例：

    sqrt(16)
log(10)
log(100, base = 10)
log10(1000)
log2(8)
exp(1)
abs(-5)
round(3.14159, digits = 2)
ceiling(3.2)
floor(3.8)
trunc(3.8)

    [1] 4
[1] 2.302585
[1] 2
[1] 3
[1] 3
[1] 2.718282
[1] 5
[1] 3.14
[1] 4
[1] 3
[1] 3

    事实上，R 中还内置了大量的科学计算函数：

    • 例如三角函数与排列组合函数等等，但是商业分析情境下使用的比较少，我们这里就不展开介绍了
    • 这些函数和功能是不需要导入任何包就可以直接使用的
    • 这点也是 R 语言的一个优势了（相比于 Python 需要导入额外的库），因此 R 会被很多学习者喜欢使用

    (2) 逻辑型¶

    (a) 逻辑型数据的定义¶

    逻辑型数据是 R 中用来表达是与否的数据类型，只有两种取值：TRUE 和 FALSE（注意全部大写）

    • 比方说我们定义两个变量 x 和 y 为逻辑型：
    x <- TRUE
y <- FALSE

typeof(x)
typeof(y)

    [1] "logical"
[1] "logical"
    • 逻辑型数据在 R 中是非常常见的，尤其是在条件判断和控制流程中，我们会经常使用逻辑型数据来表达条件的真假

    我们为什么把逻辑型分到数值型下面来讲呢：

    • 因为在 R 中，逻辑型数据和数值型数据是有对应关系的，TRUE 的数值是 1，而 FALSE 的数值是 0
    • 例如我们尝试运行以下代码：
    TRUE + TRUE
TRUE + FALSE
FALSE + FALSE
TRUE + 1
TRUE * 5
FALSE + 1

    [1] 2
[1] 1
[1] 0
[1] 2
[1] 5
[1] 1

    (b) 比较运算符¶

    逻辑型数据经常是由比较运算符来产生的

    • R 中的比较运算符包括：

      • 等于 （==）
      • 不等于（!=）
      • 大于（>）
      • 小于（<）
      • 大于等于（>=）
      • 小于等于（<=）

    • 例如我们可以运行以下代码来看看这些比较运算符的使用示例：

    1 == 1
1 == 2
1 != 2
3 > 2
3 < 2
3 >= 3
2 <= 3

    [1] TRUE
[1] FALSE
[1] TRUE
[1] TRUE
[1] FALSE
[1] TRUE
[1] TRUE

    (c) 逻辑运算符¶

    我们还可以使用逻辑运算符来连接多个逻辑表达式

    • R 中的逻辑运算符包括：

      • 与（&）：表示两个表达式都为 TRUE 时结果才为 TRUE
      • 或（|）：表示两个表达式中至少有一个为 TRUE 时结果就为 TRUE
      • 非（!）：表示取反，如果表达式为 TRUE，结果就为 FALSE；如果表达式为 FALSE，结果就为 TRUE

    • 并且，逻辑运算也是有优先级的，总得来说，括号的优先级最高，其次是非（!），最后是与（&）和或（|）

    • 例如我们可以运行以下代码来看看这些逻辑运算符的使用示例：

    1 < 5 & 2 > 1
1 < 5 | 2 > 3
!(1 < 5)
!(2 > 3)
(1 < 5 & 2 > 1) | (3 < 2)
(1 < 5 & 2 > 1) & (3 < 2)

    [1] TRUE
[1] TRUE
[1] FALSE
[1] TRUE
[1] TRUE
[1] FALSE

    (3) 空值¶

    (a) 空值的定义¶

    在 R 中，空值（NA）表示缺失值，表示数据中没有值或者值不可用

    • 这里注意一个易混淆的点：NA 不是 0，也不是空字符串 ""，它是一个特殊的值，表示缺失值
    • 例如我们可以定义一个变量 z 为 NA，来看看它的类型：
    z <- NA
typeof(z)

    [1] "logical"
    • 这里为什么 NA 的类型是逻辑型呢？这个其实算是 R 语言的不同于其他编程语言的一个特点了，R 语言中的 NA 是一个特殊的逻辑值

    (b) 空值的运算结果¶

    在 R 中，任何与 NA 进行的算数运算与比较运算都会得到 NA

    • 因为 NA 表示缺失值，我们无法确定它的具体数值，所以任何与 NA 相关的计算结果都是未知的，也就是 NA
    • 例如我们可以运行以下代码来看看 NA 的运算结果：
    NA + 1
NA * 5
log(NA)
NA == NA
NA > 0

    [1] NA
[1] NA
[1] NA
[1] NA
[1] NA

    然而，在 R 中，NA 的逻辑运算结果是有些奇怪的：

    • 我们首先看一下 NA 逻辑运算的结论：

      • NA & TRUE 的结果是 NA
      • NA & FALSE 的结果是 FALSE - 如果已经确定其中一个表达式为 FALSE，那么 FALSE 与任何表达式的逻辑与结果都是 FALSE，所以 NA & FALSE 的结果是 FALSE
      • NA | TRUE 的结果是 TRUE - 如果已经确定其中一个表达式为 TRUE，那么 TRUE 与任何表达式的逻辑或结果都是 TRUE，所以 NA | TRUE 的结果是 TRUE
      • NA | FALSE 的结果是 NA
      • !NA 的结果是 NA
      • NA & NA 的结果是 NA
      • NA | NA 的结果是 NA

    • 我们可以运行以下代码来看看这些逻辑运算的结果：

    NA & TRUE
NA & FALSE
NA | TRUE
NA | FALSE
!NA
NA & NA
NA | NA

    [1] NA
[1] FALSE
[1] TRUE
[1] NA
[1] NA
[1] NA
[1] NA

    2. 字符型（character）¶

    字符型数据，就相当于是 R 中的文字数据，它是由一系列字符组成的文本数据。

    • 在 R 中，字符型数据是由双引号 " 或单引号 ' 包裹起来的文本数据：

    • 例如我们可以定义以下几个变量来看看它们的类型：

    str1 <- "Hello, World!"
str2 <- 'R is great!'
typeof(str1)
typeof(str2)

    [1] "character"
[1] "character"

    关于字符型数据，我们在后续的课程中会详细介绍它的相关操作，这里我们先简单介绍一下。

    3. 因子型（factor）¶

    因子型数据是在字符型的基础上，给字符型数据添加了一个分类与排序属性的特殊数据类型

    • 比方说，我们如果在 R 中定义了 "一等奖"、"二等奖"、"三等奖" 这三个类别的字符型数据
    • 这三种类别之间是有一个自然的排序关系的，"一等奖" > "二等奖" > "三等奖"
    • 然而，字符型数据是没有排序关系的，它们只是简单的文本数据，无法表达这种排序关系
    • 因此，我们可以把这些字符型数据转换成因子型数据，并且指定类别的顺序

    在 R 中，我们可以使用 factor() 函数来创建因子型数据：

    • 在当中我们要强调因子类型、因子水平、以及有序因子的概念
    • 例如我们将 "一等奖"、"二等奖"、"三等奖" 这三个类别的字符型数据转换成因子型数据，并且指定类别的顺序：

    prizes <- factor(c("一等奖", "二等奖", "三等奖"), levels = c("三等奖", "二等奖", "一等奖"), ordered = TRUE)
prizes

    [1] 一等奖 二等奖 三等奖
Levels: 三等奖 < 二等奖 < 一等奖

    关于因子型数据，我们在后续的课程中会详细介绍它的相关操作，这里我们先简单介绍一下。

    4. 日期型（Date）¶

    日期型数据是在 R 中用来表示日期的数据类型，它是由年、月、日、时、分、秒、毫秒组成的特殊数据类型。

    在 R 中，我们可以使用 as.Date() 函数与 as.POSIXct() 函数来创建日期型数据：

    • as.Date() 函数用来创建只包含年、月、日的日期型数据，并且默认使用 UTC 时区
    • as.POSIXct() 函数用来创建包含年、月、日、时、分、秒的日期型数据，并且默认使用系统的本地时区
    • 例如我们可以创建以下几个日期型数据来看看它们的类型：
    date1 <- as.Date("2024-01-01")
date2 <- as.Date("2024-06-30 09:20:15")
date3 <- as.POSIXct("2024-06-30 09:20:15")
date4 <- as.POSIXct("2024-12-31 08:46:30.123")

date1
date2
date3
date4

typeof(date1)
typeof(date2)
typeof(date3)
typeof(date4)

    [1] "2024-01-01"
[1] "2024-06-30"
[1] "2024-06-30 09:20:15 AEST"
[1] "2024-12-31 08:46:30 AEDT"
[1] "double"
[1] "double"
[1] "double"
[1] "double"

    在上面的运行结果中，我们可以看到：

    • as.Date() 函数只具体到日，如果输入的字符串中包含时、分、秒部分，那么这些部分会被自动截断
    • as.POSIXct() 函数具体到秒，如果输入的字符串中包含毫秒部分，那么毫秒部分会被自动截断

    在 R 中，如果想要具体到毫秒的日期型数据，我们需要一个全局设置 options(digits.secs = 3)

    • 这个设置会让 R 语言在处理日期型数据时保留毫秒部分
    • 并且这个设置会影响到当前 R 会话中的所有日期型数据的显示和处理方式
    options(digits.secs = 3)
date5 <- as.POSIXct("2024-12-31 08:46:30.123")
date5

    [1] "2024-12-31 08:46:30.123 AEDT"
    • 如果想要取消这个设置，我们可以把 digits.secs 的值设置回默认值 0：
    options(digits.secs = 0)
date6 <- as.POSIXct("2024-12-31 08:46:30.123")
date6

    [1] "2024-12-31 08:46:30 AEDT"

    还有一个比较重要的注意事项：

    • 为什么 as.Date() 和 as.POSIXct() 函数创建的日期型数据的类型都是 "double" 呢？
    • 这是因为在 R 中，日期型数据是以一个数字来表示的，这个数字就是时间戳
    • 时间戳表示从 1970-01-01 到当前日期的秒数，所以它们的底层数据类型是 "double"，而不是我们直观上理解的 "Date" 类型
